Qor Analyzer User Guide

Qor Analyzer provide specific/detail QoR profiler/analysis via different dimensions.

Provide easy to understand statistics/charts which helps to prove Good/Bad QoR result.

Narrow down QoR outlier quickly.

Visualize hidden/potential QoR issue.

Main function of QoR Analyzer

1. Profile QoR info with specific pattern onto visible, comparable charts with dynamic data display.

2. Analyze QoR info from different dimensions and return consolidated statistics.

1. Profile Functionality

With given Log path or Flow dir, plot patterns extracted from log(s) to Charts.

1.1 Entry steps

Select log or flow mode in home page.

1.1.1 Provide valid input for log mode:

Absolute path of a log file like: /remote/us01home40/phyan/star/9001137153/base/base.log

Multiple inputs supported

1.1.2 Provide valid input for flow mode:

Absolute path of a PRS flow dir like: /remote/platform_pv1/platform_prs/PRS_2017.09-SP1/2017-08-14/DCRT2ICCII_ESPG_NOPLACE

1.2 Profile page

1.2.1 log mode

part 1 pattern tabs, multiple default QoR patterns could be extracted from log

part 2 QoR convergence chart, each is a clickable point

part 3 Share link, others could see the same page with via link

part 4 QoR compare table, compare all QoR metrics of last 2 selected steps/points

part 5 Inner-link, jump to selected chart directly

part 6 Chart check-box, could hide/display selected chart. (eg. If you want to move WNS/AREA charts closely to see difference.)

1.2.2 flow mode

Besides sections in log mode, there're additional selection under flow mode

User needs to:

1. Select a stage

2. Select a design or a metrics

part 7 Select a stage, 5 PRS stages supported: full_flow/nwpopt/nwcopt/nwropt/nwfopt

part 8 Select a single design to plot QoR info under specified stage

part 9 Select a metrics of a pattern for all designs under specified stage

1.2.3 User pattern support

User can provide regularized script to QoR Analyzer if default patterns cannot extract QoR info user focused on

What to provide:

Abolute script path (csh, perl) which could be executed directly and need to accept 1 logfile as input

Regularization:

QoR Analyzer will use output which user script prints as input and do infomation collection and translate into QoR matrix

Required printout format is:

1. print all columns names splited by space in 1st line (eg. columnA columnB columnC columnD)

2. then later lines should provide columns data with a step name. (eg. step1 0.3 0.5 0.6 1.2) (if no step name, QoR Analyzer will use default step naming rule)

So whole printout would be like below snapshot

Finally QoR Analyzer will plot the statistics provided by user script onto front-end charts together with default patterns

2. Analysis Functionality

Analyze QoR statistics under given analysis scenario

2.1 Single step QoR benefir analysis (SQB)

2.1.1 Usage scenario introduction

QoR Analyzer could provide OOTB QoR data analysis, for data inside flow, not comparing final QoR result from report only.

Analysis atom could be a single step (AREC/SIZE2/LGL, etc) during the flow, compare QoR between atom steps.

Live usage scenario 1: user wants to see QoR change after DTDPLGL step

Live usage scenario 2: new feature "Rebuffering" will introduce new step "REBUFFER", user wants to see the QoR benefit/impact after the step

2.1.2 Analysis method

Compare the QoR result between selected step and its previous step for all designs under specified flow, display Improve/Degrade distribution per metrics

There will be multiple compare sample collections in one single design. (eg. 'DRC' step will happen after 'AREC', 'LGL' or 'POWER', so analyze DRC step will collect DRCvsAREC, DRCvsLGL, DRCvsPOWER 3 compare samples)

2.1.3 How to use/where to trigger SQB?

1. Click the step of specified flow you want to analyze

2. Choose 'SQB' analysis entrance and click 'start'

2.1.4 Analysis result

Analysis result will return 2 parts:

Improve/Degrade distribution bar chart

Detail QoR change statistics table

User can switch metrics to see QoR change in different metrics or see entire QoR change on all metrics under Summary page

User can also highlight designs which he/she want to focus

2.2 Compare QoR between steps (SQC)

2.2.1 Usage scenario introduction

SQC is similar as SQB, but user needs to specified 2 steps and QoR Analyzer will compare QoR between user specified steps and exhibit the statistics.

Besides, SQC could compare same step between different flows. (Currently only support steps in PREROUTE_STG and FULL_FLOW_PPA pattern)

2.2.2 How to use/where to trigger SQC?

Compare 2 steps in same flow: click 2 step points of specified flow you want to analyze

Or compare 1 step between 2 flows: click 1 step under PREROUTE_STG/FULL_FLOW_PPA pattern tab

Choose 'SQC' analysis entrance and click 'start'

2.2.3 Analysis result

Same as SQB but only selected steps compare collection will be recorded in statistics matrix

2.3 Filter trajectory via threshold (SQT)

2.3.1 Usage scenario introduction

As per flow point of view, we cannot scan all designs plot to see whether there's QoR trajectory occurred or not

Here we provide automatice trajectory/outlier search to user to filter out steps collections where QoR change is over threshold percentage (like >20%)

2.3.2 Analysis method

Traverse steps sequentially and compare step QoR with its previous step. (eg. step A->B->C->D, will compare BvsA, CvsB, DvsC) If there's QoR change on any of the metrics is over threshold, the steps collection will be recorded into matrix

2.3.3 How to use/where to trigger SQC?

Click any points of specified flow, choose 'SQT' and enter the threshold (default 10%) you want to filter, then click 'start'

Choose 'SQC' analysis entrance and click 'start'

2.3.4 Analysis result

Same as SQB but only steps collection with QoR change over threshold will be recorded into statistics matrix
